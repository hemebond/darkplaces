497/*
498PROTOCOL_DARKPLACES3
499server updates entities according to some (unmentioned) scheme.
500
501a frame consists of all visible entities, some of which are up to date,
502often some are not up to date.
503
504these entities are stored in a range (firstentity/endentity) of structs in the
505entitydata[] buffer.
506
507to make a commit the server performs these steps:
5081. duplicate oldest frame in database (this is the baseline) as new frame, and
509 write frame numbers (oldest frame's number, new frame's number) and eye
510 location to network packet (eye location is obsolete and will be removed in
511 future revisions)
5122. write an entity change to packet and modify new frame accordingly
513 (this repeats until packet is sufficiently full or new frame is complete)
5143. write terminator (0xFFFF) to network packet
515 (FIXME: this terminator value conflicts with MAX_EDICTS 32768...)
516
517to read a commit the client performs these steps:
5181. reads frame numbers from packet and duplicates baseline frame as new frame,
519 also reads eye location but does nothing with it (obsolete).
5202. delete frames older than the baseline which was used
5213. read entity changes from packet until terminator (0xFFFF) is encountered,
522 each change is applied to entity frame.
5234. sends ack framenumber to server as part of input packet
524
525if server receives ack message in put packet it performs these steps:
5261. remove all older frames from database.
527*/

530PROTOCOL_DARKPLACES4
531a frame consists of some visible entities in a range (this is stored as start and end, note that end may be less than start if it wrapped).
532
533these entities are stored in a range (firstentity/endentity) of structs in the entitydata[] buffer.
534
535to make a commit the server performs these steps:
5361. build an entity_frame_t using appropriate functions, containing (some of) the visible entities, this is passed to the Write function to send it.
537
538This documention is unfinished!
539the Write function performs these steps:
5401. check if entity frame is larger than MAX_ENTITYFRAME or is larger than available space in database, if so the baseline is defaults, otherwise it is the current baseline of the database.
5412. write differences of an entity compared to selected baseline.
5423. add entity to entity update in database.
5434. if there are more entities to write and packet is not full, go back to step 2.
5445. write terminator (0xFFFF) as entity number.
5456. return.
